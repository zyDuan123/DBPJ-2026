 # DBPJ-2026

校园活动报名系统 
## 1. 项目简介
本项目为复旦大学《数据库系统》荣誉课程实践项目 。我们设计并实现了一个以数据库为核心的Web应用系统，旨在解决校园内活动发布、场地预约及学生报名的全流程管理问题 。系统实现了从需求分析、ER图设计、关系模式转换到3NF规范化分析的完整数据库开发流程。

## 2. 技术栈
* **前端**: Vue 3
* **后端**: Spring Boot 4 
* **数据库**: MySQL 8.4.8 LTS 

## 3. 数据库设计概况
本项目核心数据库包含6张关系表，涵盖了一对多（1:N）和多对多（M:N）关系：
* **用户表 (User)**: 采用单表继承策略，整合学生、组织者与管理员。
* **校区表 (Campus) & 场地表 (Venue)**: 实现地理位置的层级管理。
* **分类表 (Category)**: 活动类型的字典表。
* **活动表 (Activity)**: 核心业务表，关联了场地、分类及发布/审核人员。
* **报名记录表 (Registration)**: 处理学生与活动之间的多对多关系，并包含状态控制逻辑。

## 5. 核心设计亮点 (4月30日定稿阶段)
* **物理完整性**: 严格定义了主键与外键，保证数据的引用完整性。
* **业务约束**: 在数据库层实现了 `CHECK` 约束（如人数上限必须大于0）及 `UNIQUE` 约束。
* **反范式权衡**: 为优化“活动墙”高频查询性能，在 `Activity` 表中冗余存储了 `current_enrollment`（当前报名人数）字段，通过后端事务保持一致性。
* **规范化**: 整体设计满足第三范式 (3NF)，规避了插入、更新与删除异常。
